**Pick: KXBTCD + KXETHD → family `crypto`.** Reasons, in gate order:

1. **n**: 9,834 settled events per series (hourly cadence) — two orders of
   magnitude above the econ prints (38–59, hopeless for n≥300 power) and far
   above what weather accumulated in 8 months.
2. **Ladder density / fill realism**: the `KX*D` series are NATIVE cumulative
   ladders — every market is a single-leg "$X or above" threshold with its own
   two-sided book, so `ladder[boundary] = P(price > boundary)` comes from ONE
   quoted instrument (no multi-leg synthetic, unlike weather's interior
   boundaries or the KXBTC range buckets). Measured spreads 1–3c are real,
   single-leg, and exportable as an honest `half_spread` feature.
3. **Candle coverage**: settled markets return full 1-minute history (60/60
   bars on the 1-hour quote window), nearly all two-sided.
4. **Objective settlement, exactly**: `expiration_value` on every market dict
   is the realized CF Benchmarks 60s-average price — settlement against every
   boundary is exact, no bucket-midpoint representative needed.

The index families (KXINX/KXNASDAQ100) are credible third-family candidates
but are range-bucket books (multi-leg synthetics) with 4x fewer strikes; the
crypto threshold series dominate on every axis.

## Phase 2 — provider design (`research/lab/providers/crypto.py`)

* family `crypto`, market string `coin_price` = P(settle price > boundary).
* Sub-series like weather's cities: `BTC -> KXBTCD`, `ETH -> KXETHD`.
* Panel: native cumulative ladder from per-threshold mids (floor-only
  thresholds direct; cap-only complements, still single-leg), implied price
  `mid` by 0.5-crossing inversion, `final_total`/`final_margin` = exact
  `expiration_value`, `duration_sec` = quote-window length (~3,540s),
  measured single-leg `features["half_spread"]`, `stale_min`.
* `home_team = away_team = "<ASSET>H<hh>"` (asset x hour-of-day, e.g.
  `BTCH19`): the decisive team-concentration / cluster-knockout gates need a
  meaningful cluster key; for a single-underlying hourly family that key is
  hour-of-day (does the edge live only at one hour?), mirroring weather's
  city. 24 groups per asset.
* Cache `market_data/crypto/_cache/<event_ticker>.pkl` (gitignored), ATOMIC
  writes (tmp + `os.replace`), idempotent/resumable build; candles fetched
  only for markets with `volume_fp > 0` (outcome-blind activity filter),
  capped at the top 48 by volume.
* Splits `market_data/crypto/splits.json`, chronological 70/15/15 by
  (date, hour), locked; TEST never surfaced unless explicitly requested.
* `feature_store.join_panel(panel, "crypto")` at the end of `load_panel`.
* Registered in `providers/__init__._BUILTINS` (lazy import preserved).

Backfill plan (cap ≈ 10–20 API calls/s total): newest KX-era events,
stride 2 on BTC (~800 events spanning ~66 days) + stride 8 on ETH (~200
events, same calendar span), 3 sharded workers. Stride is outcome-blind
calendar widening (more walk-forward months for the same call budget).
